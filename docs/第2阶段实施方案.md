### 任务2.2.2：建立模块分层规范

#### 分层架构定义

```
┌─────────────────────────────────┐
│ L1: API层 (yl-api)               │
│ 职责：定义对外API，请求响应      │
└──────────────┬────────────────────┘
               ↓
┌─────────────────────────────────┐
│ L2: 应用层 (yl-java-web)          │
│ 职责：控制层、业务编排、协调      │
└──────────────┬────────────────────┘
               ↓
┌─────────────────────────────────┐
│ L3: 业务层 (yl-liteflow)          │
│ 职责：业务逻辑实现、核心算法      │
└──────────────┬────────────────────┘
               ↓
┌─────────────────────────────────┐
│ L4: 基础设施层                    │
│ 包含：yl-redis, yl-spit-db       │
│ 职责：数据访问、缓存、消息队列    │
└──────────────┬────────────────────┘
               ↓
┌─────────────────────────────────┐
│ L5: 通用层 (yl-common-*)          │
│ 包含：yl-common-util, exception  │
│ 职责：工具、常量、异常处理        │
└─────────────────────────────────┘
```

#### 依赖规则

- ✅ 允许：上层依赖下层
- ✅ 允许：同层模块间依赖（无向无环）
- ❌ 禁止：下层依赖上层（循环依赖）
- ❌ 禁止：跨层依赖

#### 每层职责说明

1. **API层**（可选）：定义对外服务接口
2. **应用层**：Web控制层、请求处理、路由转发
3. **业务层**：业务逻辑实现、核心算法
4. **基础设施层**：数据库、缓存、消息队列、外部服务
5. **通用层**：工具类、异常、常量

**验收标准**：
- [ ] 所有模块都清楚自己所在的层级
- [ ] 依赖方向都遵循分层规则
- [ ] 无跨层依赖
