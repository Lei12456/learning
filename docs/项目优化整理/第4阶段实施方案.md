#### 详细内容大纲

**1. 命名规范**
```
1.1 类命名 (UpperCamelCase)
  - Service/ServiceImpl
  - Controller
  - Repository
  - Util
  - Exception
  - Entity/DTO
  - Enum
  
1.2 方法命名 (lowerCamelCase)
  - get/set开头
  - is/has开头（boolean返回）
  - do/execute开头（执行操作）
  
1.3 常量命名 (UPPER_SNAKE_CASE)
  - 静态常量
  - 魔法数字
  
1.4 包命名
  - com.yl.learning[.module][.functionality]
  - controller, service, repository, entity, dto等
  
1.5 文件命名
  - Java文件与类名保持一致
```

**2. 代码格式**
```
2.1 缩进
  - 4空格或1个tab（推荐4空格）
  
2.2 换行
  - 单行代码 < 120字符
  - 方法参数超长时换行
  
2.3 空格
  - 运算符周围
  - 逗号后
  - if/for/while后
  
2.4 花括号
  - Java编码风格（不单独起行）
  - 空语句块可以简化为{}
```

**3. 注释规范**
```
3.1 类注释
  - 说明类的职责
  - 使用示例
  - @author @version @since
  
3.2 方法注释
  - 说明功能
  - @param @return @throws
  
3.3 行注释
  - 解释复杂逻辑
  - 不能注释掉代码
  
3.4 特殊标记
  - TODO
  - FIXME
  - HACK
```

**4. 异常处理规范**
```
4.1 异常捕获
  - 捕获具体异常
  - 避免吞掉异常
  - 记录异常日志
  
4.2 异常抛出
  - 使用自定义异常
  - 异常信息清晰
  
4.3 异常日志
  - 包含上下文信息
  - 记录堆栈跟踪
```

#### 具体内容

**1. 架构概览**

```
架构目标：
- 高内聚、低耦合
- 模块独立、易复用
- 清晰的分层结构
- 便于团队协作和维护

现状：
- 19个特性模块
- 清晰的模块边界
- 部分模块间有依赖关系
```

**2. 模块分层规范**

```
L5: 通用层 ────────────────────────────────────
    ├── yl-common-util      (工具类)
    └── yl-common-exception (异常处理)

L4: 基础设施层 ─────────────────────────────────
    ├── yl-redis            (缓存)
    ├── yl-spit-dataBase    (数据库)
    ├── yl-rabbitMq-producer (消息生产)
    └── yl-rabbitMq-consumer (消息消费)

L3: 业务层 ──────────────────────────────────
    ├── yl-liteflow         (流程编排)
    ├── yl-third-party-api  (API集成)
    └── yl-spring-ai        (AI集成)

L2: 应用层 ──────────────────────────────────
    ├── yl-java-web         (Web框架)
    ├── yl-xxl-job          (定时任务)
    └── yl-transactional    (事务处理)

L1: 学习示例层 ──────────────────────────────
    ├── yl-design-pattern   (设计模式)
    ├── yl-juc              (并发编程)
    ├── yl-jvm              (JVM知识)
    ├── yl-algorithm        (算法)
    ├── yl-source           (源码学习)
    ├── yl-spi              (SPI机制)
    ├── yl-spring-framework (Spring框架)
    └── yl-ai-code          (AI代码)

依赖方向：上层可以依赖下层，同层可以相互依赖，下层不能依赖上层
```

**3. 依赖规则**

```
✅ 允许：
  - 上层模块依赖下层模块（单向）
  - 同层模块间相互依赖（无环）
  - 多个上层模块依赖同一个下层模块

❌ 禁止：
  - 下层模块依赖上层模块（会形成循环）
  - 跨层依赖
  - 循环依赖结构

验证方法：
  - 运行: mvn dependency:analyze
  - 检查: maven-dependency-plugin
  - 工具: JArchitect, Structure101
```

**4. 包结构规范**

```
标准包结构（以 yl-java-web 为例）：

src/
├── main/
│   ├── java/com/yl/javaweb/
│   │   ├── controller/      (入口层，处理HTTP请求)
│   │   ├── service/         (业务名层，核心逻辑)
│   │   ├── repository/      (持久层，数据访问)
│   │   ├── entity/          (实体类)
│   │   ├── dto/             (数据传输对象)
│   │   ├── config/          (配置类)
│   │   ├── constant/        (常量)
│   │   ├── util/            (工具类)
│   │   ├── exception/       (异常类)
│   │   └── interceptor/     (拦截器)
│   └── resources/
│       ├── application.yml
│       └── logback-spring.xml
└── test/
    └── java/com/yl/javaweb/
        └── [对应的测试代码]
```
