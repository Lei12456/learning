## 📋 第3.1：代码审查和重构规范

### 任务3.1.1：建立命名规范统一标准

#### Java命名规范

**3.1.1.1 类命名 - UpperCamelCase**

```java
✅ 好的示例：
public class UserController { }         // 控制类
public class UserService { }            // 业务类
public class UserRepository { }         // 数据访问类
public abstract class BaseException {}  // 抽象类
public interface IUserService { }       // 接口（可选I前缀）

❌ 错误示例：
public class user_controller { }        // 不能用下划线
public class usercontroller { }         // 不能全小写
```

**3.1.1.2 方法命名 - lowerCamelCase**

```java
✅ 好的示例：
public void getUserById(Long id) { }
public List<User> queryAllUsers() { }
public boolean isUserExists(Long id) { }
public void doSomething() { }
public void initializeResources() { }

❌ 错误示例：
public void get_user_by_id() { }        // 不能用下划线
public void GetUserById() { }           // 第一个字母不能大写
public void GETUSERBYID() { }           // 不能全大写
```

**3.1.1.3 常量命名 - UPPER_SNAKE_CASE**

```java
✅ 好的示例：
public static final int MAX_POOL_SIZE = 100;
public static final String USER_TABLE_NAME = "user";
public static final long DEFAULT_TIMEOUT = 30000L;

❌ 错误示例：
public static final int maxPoolSize = 100;  // 不能用camelCase
public static final String Max_Pool_Size = "x"; // 不能混合
```

**3.1.1.4 属性命名 - lowerCamelCase**

```java
✅ 好的示例：
private String userName;
private Integer userId;
private List<String> userEmails;
protected boolean isActive;

❌ 错误示例：
private String user_name;              // 不能用下划线
private String UserName;               // 第一个字母不能大写
```

**3.1.1.5 包命名规范**

```
规范：com.{company}.{project}.{module}.{functionality}

示例：
com.yl.learning.javaweb.controller   // Web控制层
com.yl.learning.javaweb.service      // 业务服务层
com.yl.learning.javaweb.repository   // 数据访问层
com.yl.learning.javaweb.entity       // 实体类
com.yl.learning.javaweb.dto          // 数据传输对象
com.yl.learning.javaweb.config       // 配置类
com.yl.learning.javaweb.constant     // 常量类
com.yl.learning.javaweb.util         // 工具类
com.yl.learning.javaweb.exception    // 异常类
com.yl.learning.javaweb.interceptor  // 拦截器
```

#### 常见优化手法

**3.1.2.1 提炼方法**

```java
// ❌ 优化前：方法过长
public void processUser(User user) {
    // 参数验证（5行）
    if (user == null) throw new IllegalArgumentException();
    if (user.getId() == null) throw new IllegalArgumentException();
    
    // 数据加载（5行）
    List<Order> orders = orderService.findByUserId(user.getId());
    Map<String, Object> context = new HashMap<>();
    context.put("user", user);
    context.put("orders", orders);
    
    // 业务处理（10行）
    int totalAmount = 0;
    for (Order order : orders) {
        totalAmount += order.getAmount();
    }
    
    // 结果保存（5行）
    user.setTotalSpent(totalAmount);
    userRepository.save(user);
}

// ✅ 优化后：通过提炼方法
public void processUser(User user) {
    validateUser(user);
    Map<String, Object> context = loadUserContext(user);
    int totalAmount = calculateTotalAmount(user.getId());
    updateUserWithTotal(user, totalAmount);
}

private void validateUser(User user) {
    if (user == null) throw new IllegalArgumentException();
    if (user.getId() == null) throw new IllegalArgumentException();
}

private Map<String, Object> loadUserContext(User user) {
    List<Order> orders = orderService.findByUserId(user.getId());
    Map<String, Object> context = new HashMap<>();
    context.put("user", user);
    context.put("orders", orders);
    return context;
}

private int calculateTotalAmount(Long userId) {
    List<Order> orders = orderService.findByUserId(userId);
    return orders.stream().mapToInt(Order::getAmount).sum();
}
```

**3.1.2.2 嵌套深度优化**

```java
// ❌ 优化前：深层嵌套
if (user != null) {
```

### 任务3.1.3：异常处理规范化

#### 规范要求

**3.1.3.1 异常捕获规则**

```java
// ❌ 不推荐：捕获通用Exception
try {
    // 业务代码
} catch (Exception e) {
    log.error("发生错误", e);
}

// ✅ 推荐：捕获具体异常
try {
    // 业务代码
} catch (IOException e) {
    log.error("IO异常", e);
} catch (SQLException e) {
    log.error("数据库异常", e);
} catch (BusinessException e) {
    log.warn("业务异常", e);
}
```

**3.1.3.2 异常信息清晰**

```java
// ❌ 信息不清晰
throw new RuntimeException("错误");

// ✅ 信息清晰
throw new BusinessException("用户ID为空或不存在，无法查询订单", "USER_NOT_FOUND");
```

**3.1.3.3 异常日志记录**

```java
/**
 * 标准的异常记录格式
 */
try {
    // 操作
} catch (SpecificException e) {
    // 记录错误日志，包含：发生位置、错误原因、错误代码
    log.error("查询用户详情异常 [userId:{}]", userId, e);
    // 抛出业务异常或转换后的异常
    throw new BusinessException("查询用户失败", e);
}
```

#### 执行步骤

1. **为所有模块添加异常处理**
   - 添加必要的try-catch块
   - 添加详细的异常日志

2. **检查异常信息**
   - 确保异常信息有意义
   - 包含足够的上下文信息

**验收标准**：
- [ ] 所有异常都被正确捕获
- [ ] 异常信息清晰有意义
- [ ] 所有异常都有日志记录

```java
// ❌ 问题：重复验证逻辑

    if (username == null || username.isEmpty()) {
        throw new IllegalArgumentException("用户名不能为空");
    }
    if (password == null || password.isEmpty()) {
        throw new IllegalArgumentException("密码不能为空");
    }
    // ... 认证逻辑

    }
    if (password == null || password.isEmpty()) {
        throw new IllegalArgumentException("密码不能为空");
    }
    // ... 验证逻辑

// ✅ 解决方案：提炼验证方法
private void validateCredentials(String username, String password) {

        throw new IllegalArgumentException("用户名不能为空");
    }
    if (password == null || password.isEmpty()) {
        throw new IllegalArgumentException("密码不能为空");
    }
}

public boolean authenticateUser(String username, String password) {
    validateCredentials(username, password);
    // ... 认证逻辑
}

    validateCredentials(username, password);
    // ... 验证逻辑
}
```

#### 测试示例

```java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {
    
    @Mock
    private UserRepository userRepository;
    
    @InjectMocks
    private UserService userService;
    
    @Test
    void testGetUserById_Found() {
        // Arrange
        Long userId = 1L;
        User expectedUser = new User(userId, "张三");
        when(userRepository.findById(userId)).thenReturn(Optional.of(expectedUser));
        
        // Act
        User actualUser = userService.getUserById(userId);
        
        // Assert
        assertThat(actualUser).isEqualTo(expectedUser);
        verify(userRepository, times(1)).findById(userId);
    }
    
    @Test
    void testGetUserById_NotFound() {
        // Arrange
        Long userId = 999L;
        when(userRepository.findById(userId)).thenReturn(Optional.empty());
        
        // Act & Assert
        assertThatThrownBy(() -> userService.getUserById(userId))
            .isInstanceOf(BusinessException.class);
    }
}
```
